# SOP-07 — Gestione non conformità

> DRAFT interno. Da rivedere con consulente ISO/IEC 17025.

**Scopo.** Rilevare, registrare e trattare le non conformità (NC) di prova,
strumenti, software o procedura.

**Responsabilità.** Chi rileva (segnalazione), Responsabile qualità (trattamento).

## Esempi di NC

- Cattura rifiutata ripetutamente (qualità insufficiente).
- Riferimento scaduto usato per errore (l'app blocca, ma va registrato).
- Profilo grading ESEMPIO usato in un report destinato a uso non interno.
- Divergenza forte software vs riferimento in validazione.
- Aggiornamento software che cambia i risultati senza ri-verifica.

## Procedura

1. Registrare la NC (descrizione, data, rilevatore, prova/strumento coinvolto).
2. Valutare impatto (prove potenzialmente interessate).
3. Azione immediata (contenimento) + azione correttiva (causa radice).
4. Verifica efficacia + chiusura.
